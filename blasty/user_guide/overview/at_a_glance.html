PHP Blasty at a glance

PHP Blasty is a YUI wrapper

The PHP Blasty aim is to get YUI working in Your application easly even for who don't know noting on Javascript. Due to this, if you want to build application with complex UI, Blasty is not for You but just a point where to start.
PHP Blasty is structured as:

  • one master class Blasty
  • one static class phpBlasty used as common tool class
  • one class for each YUI component (eg. calendar)
  • the YUI PHP Loader Utility [beta]
  • the YUI JS Loader Utility (not yet developed)
  • the language translations

PHP Blasty is dual use

PHP blasty is designed to work in two modes: 'nested' and 'stand-alone'.

Nested design

In this design You instantiate the master class Blasty and each time that you ask Blasty to load a component he nests on him that component class.
You can operate with the component through the Blasty main class, for example setting a component property: 

        ...
        require_once 'blasty/blasty.php';
        $blasty = Blasty::getInstance();
        $blasty->loadComponent('calendar');
        $blasty->setProperty('PAGES', 3);                 // It will set the calendar property
        ...
        $blasty->loadComponent('datatable');
        $blasty->setProperty('caption', 'Table caption'); // It will set the datatable property
        ...

Note: Function that dialogate with the component class (es. setProperty()) will dialogate with the last loaded component. It is possible to change the current component using $blasty->setCurrentComponent().

Note: You can access directly to the component with chaining: $blasty->calendar->setProperty().

Stand-alone design

PHP Blasty is also a class collection, one class for each YUI component that You can use lonely 

        ...
        require_once 'blasty/components/calendar.php';
        $calendar = Calendar::getInstance();
        $calendar->setProperty('PAGES', 3);               // It will set the calendar property
        ...
